Capa de Sesión

Funcion

    Establecimiento y Gestión de Sesiones
  • La capa de sesión establece, mantiene y termina sesiones de comunicación entre aplicaciones. Asegura que los procesos de comunicación se inicien y finalicen de manera ordenada y que las sesiones se mantengan durante el intercambio de datos.
    Sincronización
  • Sincroniza el intercambio de datos entre aplicaciones para asegurar que los datos se envíen y reciban en el orden correcto. También puede insertar puntos de sincronización en una sesión para que, en caso de error, los datos puedan reiniciarse desde el último punto de sincronización.
    Control de Diálogo
  • Controla el diálogo entre aplicaciones, permitiendo la comunicación en modo full-duplex (bidireccional simultáneamente) o half-duplex (bidireccional alternando). Esto asegura que las aplicaciones pueden coordinar cómo y cuándo envían y reciben datos.
    Gestión de Sesiones Multiplexadas
  • Permite que múltiples sesiones de comunicación se gestionen simultáneamente a través de una única conexión de red, usando identificadores de sesión para distinguir entre diferentes comunicaciones.
    Control de Conexión
  • Supervisa y controla las conexiones entre aplicaciones para asegurar que se mantengan activas durante el tiempo necesario y que se cierren correctamente al final de la comunicación.
    Reinicio de Sesión
  • En caso de interrupciones o fallos, la capa de sesión puede reiniciar o reanudar sesiones interrumpidas desde el último punto de sincronización o estado conocido.

Dispositivos

Nombre Funcion Imagen
Servidores de Aplicaciones Los servidores de aplicaciones gestionan las sesiones entre los clientes (usuarios) y el servidor, manteniendo la conexión durante la interacción del usuario con la aplicación. Estos servidores pueden manejar múltiples sesiones simultáneas para diferentes usuarios.
Sistemas de Autenticación Sistemas como servidores de autenticación pueden ser vistos como dispositivos o servicios que trabajan en la capa de sesión, ya que gestionan el establecimiento de sesiones seguras y autenticadas entre usuarios y sistemas. También pueden controlar el tiempo de duración de las sesiones según políticas de seguridad.
